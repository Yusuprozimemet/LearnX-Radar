## Core ideas
DuckDB is a columnar database designed for high-performance data storage and querying. It is particularly well-suited for data-intensive workflows, such as those involving large language models and multimodal AI. In the context of LMCache, DuckDB is used to cache frequently accessed data, reducing the latency and improving the overall performance of the system.

One key concept in DuckDB is its columnar storage format, which allows for efficient querying and aggregation of data. This is particularly useful in workflows involving large amounts of data, where traditional row-based databases may struggle to keep up.

Here is an example of how to use DuckDB with Lance to store and query data:
```python
import lance
import duckdb

# Create a Lance database
db = lance.Database()

# Create a DuckDB connection
conn = duckdb.connect()

# Create a table in DuckDB
conn.execute("CREATE TABLE data (id INTEGER, value TEXT)")

# Insert data into the table
conn.execute("INSERT INTO data VALUES (1, 'hello')")

# Query the data using Lance
results = db.query("SELECT * FROM data WHERE id = 1")

# Print the results
print(results)
```
