# AI Agents
## Why it's on the radar today
AI Agents have surfaced in the developer world today due to their increasing use in automating tasks, such as crafting A/B test hypotheses and experiment planners [1], and building dynamic schema and data validators [3]. Additionally, there is a growing interest in using AI in daily dev workflows [4] and avoiding common pitfalls when working with AI agents [2].

## What's being discussed
The current conversation around AI Agents revolves around their use in automation, such as:
* Building staged workflows instead of relying on a single giant agent [2]
* Using OpenClaw for judgment, n8n or Make for orchestration, and scripts and APIs for deterministic steps [2]
* Crafting A/B test hypotheses and experiment planners from scratch [1]
* Building dynamic schema and data validators in record time [3]

## What you'll learn
* How to design and implement AI Agents for automation tasks
* Best practices for building staged workflows with AI Agents
* How to use AI Agents for data validation and schema creation
* Common pitfalls to avoid when working with AI Agents

## Core ideas
When working with AI Agents, it's essential to design and implement them correctly to avoid common pitfalls. One key concept is to build staged workflows instead of relying on a single giant agent. This approach allows for more flexibility and scalability.

For example, when building an A/B test hypothesis and experiment planner, an AI Agent can be used to generate hypotheses and experiment plans based on historical data and user behavior. However, it's crucial to use a staged workflow approach, where the AI Agent is used for judgment, and other tools, such as n8n or Make, are used for orchestration.

Another important concept is to use AI Agents for data validation and schema creation. By building dynamic schema and data validators, developers can ensure that their data is accurate and consistent, and that their schema is up-to-date.

## Where it fits
AI Agents fit into the broader stack as a tool for automation and data processing. They can be used in conjunction with other tools, such as n8n or Make, to build complex workflows and automate tasks.
