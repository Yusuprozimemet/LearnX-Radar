# DuckDB
## Why it's on the radar today
DuckDB has surfaced in the developer world today due to its compatibility with the Open Lakehouse Format for Multimodal AI, as seen in the lance-format/lance GitHub repository [1]. This repository highlights DuckDB's ability to provide high-performance vector search, full-text search, random access, and feature engineering capabilities for the lakehouse. Additionally, DuckDB's presence on GitHub Trending, HN Hiring, Reddit, and Stack Overflow further solidifies its relevance in current developer discussions.

## Core ideas
DuckDB is an in-process SQL OLAP database management system designed to be fast, reliable, portable, and easy to use [3]. It provides a rich SQL dialect with support for arbitrary and nested correlated subqueries, window functions, collations, complex types, and several extensions [4]. DuckDB's simplicity allows for installation in just a few seconds, and its feature-rich design supports a wide range of industry-standard technologies, such as Parquet and JSON. For example, DuckDB can be used to query a Parquet file directly, as shown in the live demo on the DuckDB website [5].

## Where it fits
DuckDB fits into the broader stack as a high-performance analytical database system that can run everywhere, including on local machines, in the cloud, or on remote servers. Developers would reach for DuckDB when they need to perform complex analytics on large datasets, or when they want to query various data sources, such as Parquet files, JSON data, or cloud storage. DuckDB's compatibility with popular data science libraries, such as Pandas and PyTorch, makes it an attractive choice for data scientists and analysts.

## Do this in 5 minutes
To get hands-on with DuckDB, open the live demo on the DuckDB website [5] and run the following query to get the top-3 busiest train stations:
```sql
-- Get the top-3 busiest train stations
SELECT station_name, count(*) AS num_services
FROM train_services
GROUP BY ALL
ORDER BY num_services DESC
LIMIT 3;
```
This will give you a taste of DuckDB's powerful SQL dialect and its ability to query complex data sources.
